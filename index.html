Project Idea

My Idea for a project is based on mostly how I could make my time for work as a Systems Administrator easier and thus be a tool for technicians in similar role that can help make life easier. To add to this, in my ideal Job as a Network Engineer this tool would be useful as well showing that in a tech world this would be useful on both a senior level and Junior level. My Idea is to program a Raspberry pi in a way that it can be used as a network monitoring device and would look something like the below.


This would help to understand if there was something that was failing within the network and also be able to pin point the time of failure. This can act as another form of troubleshooting on both a junior and senior level.

The way that this would also be setup is to have an alerting system built into it as well. This means that if there was a failure over the weekend or outside of work hours the technician would get a text or e-mail to his or her phone or e-mail.

This would likely succeed in the real world because of that fact it targets such a large group. As of June 2017, there are 2.7 million large businesses in Australia, all of these large businesses would have an IT department and infrastructure that consists of critical servers, UPS’s, Backup servers and more. This monitoring device would be targeted at these large businesses as in these situations with large international corporations for IT to run smoothly becomes more and more important to that staff of these companies as when something fails it affects more and more people.

In my current role in the IT industry I understand how easy it can be for alerting systems to fail. The issue with this is there is often no way of knowing if they are failing until the fail occurs and it becomes critical.

These critical services would be much better managed in an environment where an alerting system is setup for critical servers and services particularly for a large business that can’t miss a beat. I would like to see this implemented as having worked in the industry for a while now it can sometimes be hard to differentiate the difference of when an alerting system is not properly functioning but being able to go to the device in question and physically see what the status of it is would be a great troubleshooting aspect to have.

The drawbacks and obstacles I can see with this would probably be the physical programming of the Raspberry pi. Having had no experience with Raspberry Pi’s before this could defiantly be a challenge for me. It would take quite a bit of research and testing to get this part right.

Another challenge would also be the alerting system, again finding out a way to program this and get it all ready to be alerting to the tech. To figure out a way to setup alerting to the tech through either e-mail or text is something I have not yet done or had experience with but would likely be done with the programming of the Raspberry Pi.

The other items required for this concept would also require a centred online system with details of the servers or services being monitored, an example of this is opmanager. This is an online database that assists with managing and monitoring of these services. This could serve as an alerting system and a location to keep information on what is happing at any point in time.

If this project idea were to succeed it would defiantly serve as a solution to the original problem. Techs working in large industry style companies would have another form of checking and troubleshooting issues with their critical services when an issue arises.
